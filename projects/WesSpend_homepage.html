WesSpend Personal Stats

Budgeting App User Statistics

Tool — JavaScript · HTML/CSS
Year — 2025

Overview

This is a part of WesSpend, a software development project where I made a fully functioning cross-platform, full-stack application for student meal plan budgeting. Part of this page included a personal statistics page so that users could see their spending and how on track they were with their budget.

What I found

I chose these graphics in order to display where users were mostly spending meal points, their cumulative spending over the month, and how on track they were with their daily budget every day. A pie chart worked the best for the first graphic as it was quite easy to see where the money was being spent and to interact with the chart. Combination graphics worked best for the following graphics, because it was easy to compare them to a budget that the user could set themselves. This was a demo and didn't include any user data to test on, so there weren't any significant findings.

Process

This app implemented the Google Charts API to display interactive visualizations. The frontend of the app was built in HTML/CSS and JavaScript, so it was implemented through JavaScript and corresponds to data given through the PHP backend. It uses a RESTful API to update the data securely from changes in user input.
